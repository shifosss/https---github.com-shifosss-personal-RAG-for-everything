# contextd v0.1 — 90-Second Demo Script

Target: 90s flat, 1080p / H.264, no pauses, no typos. Re-record until clean. Final artifact lives at `docs/demo/v0.1.mp4` or is linked from the README if hosted externally.

The goal of the demo is to land a single claim: **same corpus, any agent, local-first.** Every beat below exists to reinforce one of those three.

---

## Setup (don't record)

1. Fresh macOS or WSL2 terminal in a directory that is **not** a contextd checkout (`cd ~`).
2. `pipx uninstall contextd` if a prior install exists.
3. `rm -rf ~/.contextd` — clean data home.
4. Stage `~/papers/` with ~40-50 PDFs (any real research folder works).
5. Stage `~/claude-exports/2026-03.json` (export from [claude.ai](https://claude.ai) → Settings → Data export).
6. Claude Code and Codex CLI both installed, both already configured with `contextd` as an MCP server (see README).
7. OBS or QuickTime recording at 1080p, 30 fps.

---

## 0:00 — 0:08 · Install

Cold terminal, type:

```bash
pipx install contextd
contextd --help
```

Show the subcommand list briefly, then cut.

## 0:08 — 0:30 · Ingest

```bash
contextd ingest ~/papers/ --corpus research
# Expect: "Ingested 47 sources, 812 chunks; 0 skipped, 0 failed."

contextd ingest ~/claude-exports/2026-03.json --corpus research
# Expect: "Ingested 1 source, N chunks"
```

Keep the successful summary lines on screen for ~1 beat each.

## 0:30 — 0:45 · Query from the CLI

```bash
contextd query "how did Fu handle negation" --corpus research --limit 3
```

Pan down the rich table output — source path, section label, content preview.

## 0:45 — 1:05 · Same corpus from Claude Code

Cut to a Claude Code window open on any project:

> _"Using contextd, compare how Fu and Kaster handle negation in the research corpus."_

Claude Code calls `search-corpus` via MCP, shows retrieved snippets, then composes a comparison. Linger on the tool-call trace for 2 beats so the MCP label is legible.

## 1:05 — 1:25 · Same corpus from Codex CLI

Cut to a Codex terminal:

> _"What did I conclude about Fu vs Kaster in my own notes?"_

Codex hits the same `contextd` MCP server, surfaces the Claude-export conversation. No re-ingest. No config swap.

## 1:25 — 1:30 · Closing card

Black card, three lines:

```
Same corpus. Any agent. Local-first.

contextd v0.1.0
github.com/<org>/contextd
```

## Shot list (for retakes)

- `contextd ingest ~/papers/ --corpus research` — 4s
- `contextd ingest ~/claude-exports/…` — 3s
- `contextd query …` — 6s + rich-table pan
- Claude Code prompt → answer — 20s
- Codex prompt → answer — 20s
- Closing card — 5s

Total: ~90s. If a retake is needed, re-shoot the failing beat only; the agent answers can be scripted / rehearsed so timing is consistent.

---

## Recording checklist

- [ ] No personal paths visible in prompt (`~` instead of `/Users/alex/...`).
- [ ] Terminal font ≥ 18 pt for readability on mobile.
- [ ] No API keys or secrets on screen (including in shell prompts / tmux status bars).
- [ ] Agent answers confirmed non-empty before rolling.
- [ ] Exported as H.264 MP4, ≤ 20 MB (aim lower with CRF 23).
- [ ] Captions burned in or `.vtt` sidecar provided for the closing card narration.
